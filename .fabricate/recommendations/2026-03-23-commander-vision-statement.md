# Commander's Vision Statement -- Gkestral

Date: 2026-03-23
Context: Voice-dictated during brainstorm session. Transcribed and structured by Mandan.

---

## The Core Thesis (Commander's Words)

"It is a coding CLI, but it is more than a coding CLI. It is an office tool -- like how somebody uses Microsoft Office. A new era of an office tool."

"Things are changing with AI. Most people will not be creating documents from scratch. AI will create everything for them, and then it comes into a real viewer where they will be working on it."

"This is a people tool. This is not an 'AI does everything' tool. AI is not gonna replace the student. AI is not gonna replace the individual. It is information -- working with information, access to information, utilities that help you deal with information."

---

## What Gkestral Is

A **new-era office tool** -- the AI equivalent of Microsoft Office.

Not a coding tool. Not an agent that replaces people. An **information tool for people**.

The user says:
- "Give me a DOCX" -- and out comes a document
- "Give me a PPTX" -- and out comes a presentation
- "I want a presentation for school tomorrow about X" -- the AI researches, talks to the user, produces it
- The user then modifies, refines, approves

This applies to:
- A doctor
- A researcher
- A student
- A founder
- A developer

The same tool. The same interface. The same loop.

---

## How It Differs From Others

### vs OpenClaw / Agent Frameworks
"Open Claw is about 24/7 service agents doing all your work. That is not the approach we are taking."

Gkestral is about **assisting people with information**. Not replacing them.

### vs Anthropic / Claude Desktop
"Anthropic is very good but it is more on the coding side. Desktop is only halfway there. It is still not multimodal in the same sense as Google can be."

### vs Google's Own Tools
Google has the capabilities but they're fragmented. Gkestral unifies them.

### vs Nothing (The Market Gap)
"We have that two-pane approach. One pane is a CLI where you chat with the AI. On the other side, you have all their outputs. I don't think we have a tool like that at the moment in the market."

---

## The Two-Pane Product

**Left pane:** CLI where you chat with AI. Conversation. Direction. Refinement.

**Right pane:** All outputs. Documents. Presentations. Research. Images. Maps. Code (when relevant).

This is the product. Chat on the left. Artifacts on the right. Nobody has this as a desktop tool.

---

## Infrastructure Is The Same Regardless of Use Case

Whether the user is coding or researching or writing a school report, the foundation is identical:

1. **Gemini interface** -- the AI engine
2. **Google services interface** -- Gmail, Drive, Search, Maps, Calendar
3. **Single OAuth** -- one login unlocks everything
4. **MCP skills** -- whatever is needed to access services
5. **Local workspace** -- folders on the user's computer
6. **Document generation** -- DOCX, PPTX, PDF, images (code runs behind the scenes)

A coding user uses the same infrastructure but sees the code layer. A student uses the same infrastructure but never sees code.

---

## The Business Model Insight

"The intensity of how much a person can do depends on the subscription level."

- Free Google account: basic Gemini, basic grounding, local workspace
- Paid Google account: more tokens, caching, Drive/Gmail, advanced research
- Every Gkestral user is a Gemini API consumer
- Gkestral becomes a sales channel for Google's subscription tiers

"Instead of having to go on to the Gemini portal and be limited there, he is now having access to information in his computer and saving to his workspace. A session that remains there. Instead of having to go back into the website."

---

## The "New Era Office Tool" Framing

Microsoft Office: you create documents manually. Word, Excel, PowerPoint.

Gkestral: AI creates documents for you. You review, refine, approve.

The shift: **from creation tools to direction tools.** The user directs. The AI produces. The user refines.

This is the next generation of what "office software" means.

---

## What The User Need Not Know

- Nothing about coding
- Nothing about APIs
- Nothing about Gemini models
- Nothing about prompting

They just say what they need. In natural language. In their pane. And the output appears in the other pane.
