# Stitch Prompt for gKestral Landing Page

## Prompt to paste into Stitch:

---

Create a modern, premium landing page for gKestral — a Gemini-native desktop workbench that turns scattered context into finished work.

**Brand identity:**
- Product name: gKestral (lowercase g, capital K)
- Brand mark: "gK" monogram
- Colors: deep navy/dark (#0d1117), vibrant blue (#0e55d9), teal accent (#23b4a8), white text
- Fonts: clean sans-serif (Inter or Manrope), monospace for technical elements
- Tone: calm, serious, quietly powerful. Not a toy, not a demo. A real desktop tool.
- No emojis. No playful gradients. No generic AI glow effects.

**Page structure (single scrolling page):**

**1. Hero Section**
- Headline: "Turn scattered context into finished work."
- Subtext: "gKestral is a Gemini-native desktop workbench for research, documents, visuals, and execution. One workspace. One login. One objective-completion loop."
- Two-pane product mockup showing: left pane (dark terminal-style conversation), right pane (canvas with research cards, document preview, and image)
- CTA button: "Coming Soon" (disabled/muted style)
- Small note: "Built with Go. Powered by Gemini."

**2. What It Is section**
- Three cards in a row:
  - "Information Tool" — Gather research, emails, files, and web sources into one surface
  - "Document Engine" — Turn scattered inputs into briefs, reports, presentations, and plans
  - "Execution Layer" — Code, automation, and file operations run behind the scenes when needed
- Design: dark cards with subtle borders, numbered 01/02/03

**3. The Two-Pane Design section**
- Large mockup of the desktop app showing split view:
  - Left: dark conversation pane with user message and AI response, tool call indicators
  - Right: canvas pane with tabs (Research, Code, Image, Map) showing research cards with citations
- Caption: "Chat on the left. Artifacts on the right. Everything visible."

**4. Who It Serves section**
- Three persona cards:
  - "Students & Researchers" — Source PDFs, synthesize findings, produce cited documents
  - "Founders & Operators" — Email to action, market intelligence, decision briefs
  - "Technical Builders" — Screenshot to fix, codebase analysis, architecture reasoning
- Subtle icons (no emojis), clean grid

**5. Why Gemini section**
- Four capability pills or feature blocks:
  - "1M Token Context" — Hold your entire project in memory
  - "Google Search Grounding" — Live research, cited sources, not stale training data
  - "Multimodal Input" — Screenshots, PDFs, diagrams, images as natural context
  - "Context Caching" — 90% cost reduction on repeated queries. Your workspace stays warm.

**6. What Makes It Different section**
- Comparison-style layout (not a table, more editorial):
  - "Not another coding CLI" — gKestral is an information tool. Code runs behind the scenes.
  - "Not a chatbot" — It produces artifacts: documents, presentations, code, research.
  - "Not browser-bound" — Sessions persist locally. Your workspace is folders on your computer.
  - "Not model-agnostic" — Gemini-native by design. Every feature exploits what Gemini does best.

**7. Footer**
- gKestral logo/wordmark
- "A product by Aarksee"
- Links: GitHub (coming soon), Documentation (coming soon)
- "Built with Go. Powered by Google Gemini."
- Copyright 2026

**Design requirements:**
- Dark mode only (deep navy background, not pure black)
- Generous whitespace, clean typography hierarchy
- Subtle reveal animations on scroll (fade-up)
- Fully responsive (mobile, tablet, desktop)
- No stock photography. Use UI mockups, geometric shapes, or abstract patterns only.
- The product mockup should look like a real desktop application with a window chrome (title bar, close/minimize/maximize buttons)
- Professional, investor-ready aesthetic. Think Linear, Vercel, or Raycast landing pages.
