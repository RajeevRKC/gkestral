      When a prompt is sent to gemini-2.5-flash via the Go client

      When a multi-turn conversation includes a model response with thought parts
      Then thought signatures are extracted, validated, and circulated back in subsequent requests
      And the SYNTHETIC_THOUGHT_SIGNATURE pattern is correctly handled

AC-3: Given a system prompt and conversation history exceeding 1024 tokens
      When a context cache is created with stable content
      Then the cache returns a cache name and TTL
      And subsequent requests using the cache report cached_content_token_count > 0

AC-4: Given tool declarations registered with the client
      When the model returns parallel function calls with distinct ids
      Then each function call is dispatched independently
      And results are matched back to their originating call by id

       Then gemini-3.1-flash is selected

AC-8: Given a model returns a 503 or connection reset during streaming

      Then partial content from the failed stream is discarded
      And the error is propagated to the consumer via the event channel

<task id="T6" wave="2" depends_on="T1,T2" complexity="moderate">
  <files>internal/gemini/retry.go, internal/gemini/retry_test.go</files>
  <action>
    Build retry engine with exponential backoff and user callback:

    - RetryConfig: MaxAttempts (default 10), InitialDelay (2s), MaxDelay (30s),
      JitterFraction (0.2), OnPersistent429 func(attempt int) bool
    - WithRetry(opts ...RetryOption) -- functional options for retry config
    - retryableError(err error) bool -- classify errors:
      * HTTP 429 -> retryable (rate limit)
      * HTTP 500, 502, 503 -> retryable (server error)
      * ECONNRESET, ETIMEDOUT, EPIPE, ENOTFOUND, SSL errors -> retryable
      * HTTP 400, 401, 403, 404 -> NOT retryable
    - executeWithRetry(ctx, fn) (result, error):
      * Exponential backoff: delay = min(initialDelay * 2^attempt, maxDelay)
      * Add random jitter: +/- jitterFraction * delay
      * On 429 after 5 consecutive attempts: call OnPersistent429 callback
      * If callback returns false (user says stop): return error immediately
      * Respect context cancellation between retries
    - Model fallback: FallbackChain []string -- try next model on persistent failure
      * Default chain: preview -> default tier (e.g., 3.1-pro -> 2.5-pro)

    Test: verify backoff timing, jitter distribution, callback invocation,
    context cancellation, and fallback chain behaviour.
  </action>
  <verify>go test -race ./internal/gemini/ -run TestRetry passes</verify>
  <done>Retry engine handles 429, server errors, and model fallback</done>
