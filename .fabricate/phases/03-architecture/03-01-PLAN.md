<acceptance_criteria>
AC-1: Given Wails v2 is installed
      When `wails dev` is run in the project root
      Then a native window opens with a two-pane layout
      And the left pane shows a chat input area
      And the right pane shows an empty artifact space

AC-2: Given the application is running
      When the user types a message and presses Enter
      Then the message is sent to Gemini via the streaming API
      And response tokens appear incrementally in the chat pane
      And the complete response is stored in conversation history

AC-3: Given a conversation with tool declarations registered
      When Gemini returns a function call (e.g., read_file)
      Then the tool registry dispatches to the correct handler
      And the tool result is returned to Gemini for the next turn
      And the tool execution is visible in the chat pane

AC-4: Given a conversation history with 5+ turns
      When the application is closed and reopened
      Then the previous session is loaded from SQLite
      And the conversation history is displayed in the chat pane
      And the user can continue the conversation

AC-5: Given a tool execution that returns >10,000 characters
      When the result is injected back into the conversation
      Then observation masking reduces the content to a summary
      And the full output is available on demand (expandable)
      And the context budget tracker shows token usage

AC-6: Given a task classified as "deep reasoning" by the user prompt
      When the engine processes the request
      Then gemini-3.1-pro-preview is selected with thinking enabled
      And for "fast extraction" tasks, gemini-3.1-flash is selected

AC-7: Given the user has not signed in with Google
      When the user clicks "Sign in with Google" in the application
      Then the OAuth desktop PKCE flow runs (browser opens, consent, callback)
      And tokens are stored securely
      And the user's email is displayed in the UI

AC-8: Given all internal/ packages
      When go test ./internal/... is run
      Then all tests pass (including engine, tools, workspace)
      And coverage for new packages >= 80%

AC-9: Given the engine is configured with a Gemini API key
      When a search-grounded query is sent
      Then the research tool is invoked
      And citations are extracted and available in the response

AC-10: Given the workspace model
       When the application is opened in a directory
       Then the workspace is detected (project name, type, file tree)
       And workspace metadata is available to the engine for context
</acceptance_criteria>
